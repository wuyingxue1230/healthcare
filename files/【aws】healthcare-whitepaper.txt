Cerner 的语音识别工具可以被动聆听临床医生与患者的谈话。用户可以不受干扰地畅所欲言，
而工具可以处理用户所说的内容并以此提供重要数据。该工具可根据医患对话来提供诊断、一
键订购以及潜在的过敏信息。

想像一下年长亲属从楼梯上摔下后家庭成员陪护着颤巍巍的老人前往医院急诊
室的场景。患者抱怨头痛，但情况似乎并不是特别危急。这是星期五的晚上，
当地急诊室十分繁忙。放射科医生的工作清单上已经有 85 位患者，并且可能需
要等待两个小时才能在扫描中发现这位患者极为危险的肺动脉大块栓塞。
现在，一切都改变了。经过多年将患者记录数字化并利用云技术的努力之后，
医疗保健行业创建了庞大且仍在增长的数据池。分析工具和越来越多的机器学
习所使用的数据可以推进从简化医院工作流程到促进癌症或肺栓塞早期检测的
一切工作。

“

数字化其实只是第一步。真正的力量在于如何从数字化中获
得第二阶段的效果。”
Ryan Hamilton，Cerner 公司人口健康高级副总裁

诸如 Cerner、Aidoc 和 Arterys 之类的公司正在利用 Amazon Web Service (AWS)
的高速且大容量数据存储、云中处理和检索以及机器学习工具来开发和应用机
器学习算法，从而为患者和医护人员等群体带来积极的影响。

2

机器学习算法在医疗保健领域的兴起

机器学习算法
Hamilton 说，现在已经证明机器学习对于医疗保健提供者来说是一种可行的
资源，下一步是在决策时扩展创建智力的规模并将其集成回工作流中。Cerner
自己正在构建复杂的分析工具，这些工具将利用已经获得的安全、匿名化的大
量患者数据，包括医疗诊断和治疗结果、索赔和编码产生的财务结果、计费工
具、可预测的医院人员配置模型等。例如，考虑之前提到的栓塞患者所访问的
急诊室。美国各地的设施都面临人员配备的挑战。Cerner 借助其机器学习算法
中的一种，可以利用历史数据来预测患者数量并相应地提前几天为急诊室配备
人员。这种主动算法可以确保医生和护士在轮班期间工作不会过于繁重，并且
可以更快地为患者诊治并提供优质的护理。
构成要素
Cerner 希望基于其匿名化的、符合 HIPAA 要求的记录，通过 Amazon SageMaker
快速推进机器学习，并探索其他应用。
“AWS 使我们能够获得各种工具和技术，无论它们是基本的构成要素还是复杂的
生态系统，例如 SageMaker。从历史上看，这些都是我们必须自己投资和发明
的技术。”Hamilton 说。
但 Hamilton 也表示，Cerner 无法构建市场所需的所有算法。该公司已经与合作
伙伴合作，在 Cerner 生态系统内构建机器学习模型。但是，要真正看到大规模
的影响，他认为需要更广泛的合作。

在必要时做出决策：改善诊断

幸运的是，Cerner 并不是唯一一家使用 AWS 机器学习功能来开发医疗保健模型
的公司。在机器学习模型中提高速度和准确性是该技术的令人兴奋的应用。

“

如果你能及时
为那名患者诊
治，一切都会
改善。”
Aidoc 首席执行官
Elad Wallach

回头想想那个正在等待 CT 扫描结果的老年患者。智能放射学初创公司 Aidoc 将
加快审查过程。该公司的深度学习算法利用数百万条过往记录中的知识来准确
识别不断增加的栓塞，然后立即在 ED 放射队列中的患者图像上放置一个红旗标
记。这一算法并不按时间顺序逐个审查扫描结果，而是将异常情况推到队列之
首，然后迅速将患者送往重症监护室。
Aidoc 首席执行官 Elad Wallach 表示：“如果你能及时为那名患者诊治，一切都会
改善。”Aidoc 技术现已在全球数百处设施中部署，且已经取得了可衡量的成果：
在东海岸的一家主要医疗中心，将机器学习应用于 CT 图像可将患者每天在医院
的停留时间缩短四分之三，并且将在 ED 中花费的时间缩短 59 分钟。一项类似的
前瞻性研究将颅内出血的 CT 扫描 ER 周转时间从 53 分钟减少到 46 分钟。
除了挽救患者的生命之外，医生也非常需要这种支持。 放射科人员极为有限，
大量的工作可能会导致疲劳，但各科室大约有 65％ 的医生都表示工作过于繁
重。机器学习工具可以提醒医生扫描中的异常情况，并标记可能的诊断以供医
生检查，从而为医生节约了宝贵的审阅时间。

简化医生的工作流程
另一家使用 AWS 构建和托管医护机器学习算法的开拓性的公司 Arterys 正
在基于其整个平台来帮助医生更有效地检查和诊断患者的病情。该公司花
费多年时间开发了强大的查看器界面，供医学专业人士访问医学图像、在
临床工作流程中运行 AI 模型以及进行比较分析。尽管该平台通过 AWS 使
用多种技术来改善医生的工作流程（从报告功能到高分辨率成像），但是
机器学习正在改善诊断成像过程.
使用机器学习增强诊断过程

“

将 AI 与 AWS 云的
能力相结合，世界
各地的每个人都有
机会进行早期癌症
诊断，并增加生存
机会。”

Fabien Beckers，Arterys
创始人兼首席执行官

Arterys 开始探索在新生儿和幼儿中使用机器学习执行心脏缺陷的诊断应
用。这些异常以前是通过超声波检测的，只能提供部分视图，无法计算
血流异常。该平台使用 AWS Machine Learning 和云实例，轻松完成创建
和比较有用的血流图所需的海量数据处理。心脏分析时间从 40 分钟缩短
到了 6 分钟。
Arterys 还探索了机器学习诊断在癌症，特别是肺癌（主要的致死癌症）
中的应用。从历史上看，放射科医生会不遗余力地检查扫描结果，以试
图将肺结节与癌性病变分开。使用 AWS，几乎可以立即将新图像与数十
万个类似病例进行比较来识别这些结节，从而提高准确性和工作效率。
然后，专业人员可以在系统内处理，以查看多个时间点的多个视图，并
查看结节随着时间的推移可能出现的自动进展模型。现在，全球有 100
多家医院和成像中心在其成像工作流程中无缝使用 Arterys。
全球范围内创建有成千上万的 AI 模型，并且没有将其部署到临床工作流
程中的简单方法，Arterys 平台使这些模型可以通过简单的 Web 浏览器进
行部署。Arterys 正在建立一个数据驱动、智能且以患者为中心的临床护
理世界。

机器学习为医疗保健提供了一种
整体性方法

通过 AWS 运用机器学习技术的三家公司都同意：这仅仅是开始。尽管他们已经
通过 AWS Machine Learning 达到了前所未有的规模，但要将机器学习的成功应
用到医疗的其他领域还有很多工作要做。

“

“嘿”医生发短信说，

我想您会很
高兴知道
Aidoc
刚挽救了一
名患者的
性命。”
Ed Wallach，
Aidoc 首席执行官

Hamilton 设想机器学习将为患者创造更多机会，以便其与自己的医疗保健提供
者进行富有成效的对话。该公司目前正在开发自然语言处理（NLP）技术，该技
术可以通过智能地聆听考场中的对话，使医生摆脱将数据输入笔记本电脑的麻
烦； NLP 模型可以做笔记并提取相关的测试结果或图像。后台应用程序可能会
基于数百万条记录为医生显示替代诊断。
拜访结束后，护理并不会结束：回到家，患者可能会收到交互式提示：“您是否
按照处方开了药？ 如果我们将药品寄到您家中，会对您有帮助吗？” 可穿戴技
术的远程监控可以祝贺患者达到每天3,000步，或者警告病例管理员，体重增加
过快表明有待解决的心力衰竭。
汉密尔顿说，目标是“试图进入对话式医疗工作流程，而不是面向访问的，面向
场所的工作流程。”
尽管这些可能性继续激发人们对机器学习的兴趣，但公司一致认为，迄今为止
的进展令人欣喜，尤其因为它真的有能力挽救生命。Wallach说，他一直被每周
从医生处直接收到的信息所激励，包括最近收到的有关一位患有严重肺栓塞的
ER患者的信息。
